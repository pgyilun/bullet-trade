# 邀请贡献

欢迎在 GitHub 提 Issue/PR 或通过微信交流，一起完善 BulletTrade 的文档、数据/券商适配与示例。

## 如何参与
- 反馈与问题：在仓库 Issue 中描述使用场景、配置与日志，便于复现。
- 文档/示例 PR：遵循下方检查清单，保持中文描述，必要时补充英文术语。
- 需求建议：说明业务目的与期望接口，团队会评估后纳入规划。
- 交流：可通过站点上的微信二维码联系，或在 PR/Issue 里继续讨论。

## 测试矩阵

默认快速测试不应依赖外部行情、券商软件或网络服务：

```bash
pytest -m "not slow and not requires_network"
```

带外部依赖的测试需要显式选择标记，并确认运行环境已经准备好：

- `requires_qmt`：需要本机或测试机已经安装并登录 MiniQMT/xtquant，且测试账号、行情权限和 QMT 客户端状态可用；不要在默认 CI 中运行。
- `requires_network`：需要可访问外部数据服务或远程行情服务，并已经配置相应账号或网络代理；不要把账号、密钥或私有服务地址写入测试代码和日志。
- `requires_jqdata`：需要聚宽数据源账号或兼容数据源可用，适合手动集成验证，不适合作为无凭据的快速测试。
- `slow`：用于长周期回测、跨数据源对比和性能基准，运行前应记录数据源、缓存状态、命令、耗时和核心结果。
